# IMC Prosperity 4 - Quantitative Trading Bot

This repository contains our state-of-the-art (SOTA) algorithmic trading bot for the IMC Prosperity 4 challenge (Round 1). The strategies implemented are based on rigorous quantitative research, DP bounds calculation, and grid-search parameter optimization.

## Repository Structure

The repository has been thoroughly cleaned and organized to reflect only our best models and reusable analysis tooling:

- `105934.py`: The final, optimized submission script.
- `105934.json / .log`: The official judge evaluation logs and results for the submission.
- `round1/`: Contains the original historical market data (`.csv` files) for backtesting.
- `scripts/`: A suite of reusable tools for data analysis, theoretical bounds calculation, and strategy optimization.

### Strategy Overview (`105934.py`)

Our approach tackles the macro-structural traits of the Round 1 products, deliberately avoiding micro-structural overfitting:

**1. INTARIAN_PEPPER_ROOT (Directional Drift + Spike Laddering)**
- **Market Physics:** The asset experiences a continuous, deterministic upward drift of roughly `+0.1` per tick.
- **Action:** The bot executes aggressive "Market Taking" sweeps to reach its maximum long limit (`+80`) immediately.
- **Edge:** It places a ladder of asks at extreme deviations (`+4, +8, +12` ticks away from the spread) to capture liquidity from noise traders and fat-finger spikes without risking downward whipsaws.

**2. ASH_COATED_OSMIUM (Mean Reversion + Imbalance Skewing)**
- **Market Physics:** The asset is highly stationary, continuously reverting to `10,000`.
- **Action:** A High-Frequency Mean-Reversion strategy that utilizes Level 1 Orderbook Imbalance (`imb1`) alongside a Z-score pullback to `10,000`.
- **Edge:** Grid-searched optimizations found that quoting with a highly aggressive skew (`4.0 * position / limit`) paired with a strict market-taking condition on extreme imbalance (`> 0.8`) extracts the absolute maximum PnL.

## Tooling & Scripts

The `scripts/` directory contains tools built to calculate theoretical edges and optimize live parameters.

### Data Analysis Tools
- `analyze_judge_data.py`: Parses `.json` submission logs generated by the official judge to reconstruct the true market physics and PnL curves.
- `analyze_drift.py`: Calculates absolute start-to-end price drifts for all assets over given data files.
- `analyze_trades.py`: Computes statistical bounds (Max, Min, Mean, Std Dev) of historical trades.
- `calc_max_pnl.py`: Uses Dynamic Programming (DP) to compute the absolute theoretical maximum PnL given perfect hindsight to evaluate backtester limits.

### Optimization Tools
- `opt_full.py`, `opt_pepper.py`, `optimize_osmium.py`, `optimize_optuna.py`: Multiprocessing grid-search and Optuna pipelines used to repeatedly run the backtester against varying hyperparameter sets (e.g., base edge, skew multiplier, signal scaling) to isolate globally optimal configurations.

## Local Backtesting

We utilize the `prosperity4btx` engine to simulate the exchange matching engine locally.

**Installation:**
```bash
pip install prosperity4btx
```

**Running a Backtest:**
You can test the submission against the historical data located in the `round1/` directory.

```bash
# Ensure you are at the root of the repository
prosperity4btx 105934.py 1 --data ./round1/ --match-trades worse --limit ASH_COATED_OSMIUM:80 --limit INTARIAN_PEPPER_ROOT:80
```

> Note: The `--match-trades worse` flag forces conservative matching behavior, ensuring our local optimizations are structurally sound and not simply abusing optimistic simulation conditions.
